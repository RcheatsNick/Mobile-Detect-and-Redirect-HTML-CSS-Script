How Does It Worktop

This plugin detects if the user is visiting your website from a mobile device and if that is the case it will show him a window where he can choose between two options, stay on the main website or visit the mobile version.

To prevent the window from showing up each time the user visits your website, or goes to another page this plugin uses a cookie system which remembers the user's choice, if he wants to.

The cookie script will only work on a server, so testing it locally is not possible.

How To Customize Your Itemtop

This item has two files, a JavaScript file named "js/mobile-detect-redirect.js" and a CSS file named "css/mobile-detect-redirect.css". Both file are required on the same page.

All the settings can be found in the JavaScript file at the top.

The settings look like this:

// initial settings start
	var showRedirectWindow = 'true'; // set this to 'false' to disable the redirect form, and send users directly to the mobile version
	var redirectToPage = 'false'; // set this to 'true' if you want to redirect from a page to the corresponding mobile page, e.g. www.example.com/page - m.example.com/page
	var redirectProtocol = 'http://'; // set this to  'https://' if you need to
	var redirectLocationName = 'm'; // if 'redirectFolder' is set to 'false' this value will be used as the sub-domain and the redirect URL will be 'http://m.example.com'. If 'redirect folder is set to 'true' the URL will become 'http://example.com/m'
	var redirectToFolder = 'false'; // set this to true if you are redirecting to a sub-folder and not to a sub-domain
	var redirectDomain = 'example.com'; // set your main domain
	var redirectURL = ''; // do no modify!
	var redirectTitle = 'Mobile device detected';
	var redirectMessage = 'This page is not optimized for mobile devices, would you like to visit our mobile website?';
	var redirectChoice = ''; // do not modify!
	var rememberChoice = 'false'; // do not modify!
	var redirectWindowWidth = 252; // the width of the form without paddings and borders
	var redirectWindowHeight = 280; // the height of the form without paddings and borders
	
	var redirectPath = window.location.pathname; // do not modify!
	
	var cookieLife = 7; // the number of days until the cookie is deleted
	var redirectCookie = ''; // do not modify!
	
	var mobileDevice = (/iphone|ipad|ipod|android|iemobile|windows phone|blackberry|BB10|opera mini/).test(navigator.userAgent.toLowerCase()); // you can add more mobile devices here
	
	var desktopMode = 'false'; // 'true' for tests only, the default value should be 'false'
	// initial settings end

You may also want to adjust your "viewport" settings for your main website, like the "initial-zoom" value and "user-scalable" value. If they are not set, the page might be zoomed out and fitted to the users mobile devices, making the window too small (of course in this case the user can zoom in and still be able to read the message).

Integrate This Script Into Your Projecttop

If you want to add this plugin to your website you need to add the "mobile-detect-redirect.js" JavaScript file and the "mobile-detect-redirect.css" CSS file to your HTML or PHP files (pages).

Let us assume you have a HTML template inside a folder called "website". You need to copy the two files mentioned above to that folder. You can place them in sub-folders if you wish, for example the JavaScript file "mobile-detect-redirect.js" can be placed in "website/js/mobile-detect-redirect.js" and the CSS file "mobile-detect-redirect.css" in another sub-folder like this "website/css/mobile-detect-redirect.css".

Now you need to include both files into your HTML template along with jQuery. To do that, open your HTML files in a code editor, and add the lines bellow between the "head" tags:

<link href="css/mobile-detect-redirect.css" rel="stylesheet" type="text/css" media="all" />
<script type="text/javascript" src="js/jquery.min.js"></script>
<script type="text/javascript" src="js/mobile-detect-redirect.js"></script>

The line of code for the JavaScript file does not have to be right after the CSS file, you can group them with your other files to keep your code organized.

You must include the "mobile-detect-redirect.js" file after the line of code for jQuery, otherwise it will not work.

Please make sure the paths for the files are correct. In this example we assumed the HTML files are in the main folder called "website" together with the folders "js" and "css".

You do not need to use the included HTML file "index.html", or any of its code. The "images" folder is not required.

For WordPress based websites you should use enqueue script and enqueue style.
